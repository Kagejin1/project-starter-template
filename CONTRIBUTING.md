# Contributing

## Branch naming
- `feat/<topic>`
- `fix/<topic>`
- `refactor/<topic>`
- `chore/<topic>`

## Commit convention
Use Conventional Commits:
- `feat: add X`
- `fix: resolve Y`
- `refactor: simplify Z`
- `test: add coverage for A`
- `ci: update pipeline`
- `docs: improve readme`
- `chore: maintenance`

## Pull requests
1. Keep PRs small and focused.
2. Link issue when applicable.
3. Ensure CI passes before merge.
4. Update docs when behavior changes.
